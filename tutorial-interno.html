1. Visão Geral do Sistema

O que é e como funciona

O que é o Rankings OdonConcursos

Site onde candidatos a concursos odontológicos enviam suas notas e se comparam com outros participantes. Funciona como um gabarito coletivo voluntário.

URL pública: https://odonconcursos.github.io/odon-rankings/

2. Guia do Usuário Comum USUÁRIO

O que qualquer pessoa pode fazer no site

Criar conta

  • Acessar o site e clicar em "Criar conta" no menu superior direito.
  • Escolher entre e-mail + senha ou Continuar com Google.
  • Preencher nome, e-mail e senha (mínimo 6 caracteres) se optar por e-mail.
  • Após cadastro, o login é feito automaticamente e o usuário já está dentro do site.

Fazer login

  • Clicar em "Entrar" no canto superior direito.
  • Digitar e-mail e senha, ou clicar em "Continuar com Google".
  • Após login, o avatar com as iniciais aparece no canto superior — confirma que está logado.
Senha esquecida: Clicar em "Esqueci minha senha" na tela de login → recebe e-mail de redefinição.

Navegar no site

O menu superior está presente em todas as páginas do site e contém:

  • Como usar — abre o tutorial público com o passo a passo para candidatos (tutorial-usuario.html).
  • Entrar / Criar conta — visível para quem não está logado.
  • Meu histórico / Painel Admin — visível apenas para logados (Painel Admin só para admins).

Ver rankings existentes

  • Na página inicial, todos os rankings aparecem como cards com nome da prova, banca e data.
  • Usar a barra de busca ou o filtro de bancas para encontrar uma prova específica.
  • Clicar no card para abrir o ranking completo com tabela de candidatos e notas.
💡 Não precisa de conta para visualizar rankings. Qualquer pessoa pode ver.

Criar um novo ranking

  • Na página inicial, clicar em "+ Criar novo ranking".
  • Preencher: Nome da prova/cargo, Banca organizadora e Data da prova.
  • Clicar em "Criar ranking" — o site redireciona para o ranking criado (vazio).
💡 Qualquer pessoa pode criar um ranking — não precisa de conta.

Enviar nota em um ranking

  • Abrir o ranking da prova desejada e clicar em "Enviar minha nota".
  • Se não estiver logado, o site redireciona automaticamente para a tela de login com uma mensagem explicativa. Após o login, o usuário é devolvido ao formulário de envio.
  • Preencher o nome de exibição (pode marcar "apenas minhas iniciais" para privacidade).
  • Digitar a nota total (obrigatório) e acertos por disciplina (opcional).
  • Anexar o PDF ou imagem da prova via campo "Enviar prova" (opcional).
  • Marcar a opção "Sou aluno(a) OdonConcursos" se for aluno(a) — aparece o badge 🎓 Aluno(a) Odon no ranking.
  • Clicar em "Enviar nota" — ranking atualiza na hora com a nova posição.
💡 Badge Aluno(a) Odon: O campo alunoOdon: true/false fica salvo no Firestore junto com a nota. O badge rosa aparece ao lado do nome no ranking para facilitar a identificação dos alunos da OdonConcursos.
💡 Envios anônimos anteriores: Notas enviadas antes de 05/05/2026 (quando o login ainda era opcional) foram preservadas e continuam visíveis no ranking. Porém, o dono não consegue mais editá-las sem criar uma conta.

Editar nota já enviada

  • Abrir o ranking da prova onde enviou a nota.
  • Clicar no botão "Editar minha nota" (visível apenas para quem enviou).
  • Alterar os campos desejados e clicar em "Salvar alterações".
💡 Para editar é preciso estar logado com a mesma conta usada ao enviar a nota.

Ver histórico pessoal

  • Clicar em "Meu histórico" no menu superior (só aparece para logados).
  • Ver resumo geral: provas participadas, melhor posição, maior nota, média.
  • Lista de todas as provas com posição e nota em cada uma.

3. Guia do Administrador ADMIN

Exclusivo para a equipe OdonConcursos

Acessar o Painel Administrativo

  • Fazer login com um dos e-mails admin (michelecostaor, adm.odonconcursos ou odonconcursosvendas).
  • Após o login, o botão "Painel Admin" aparece automaticamente no menu superior.
  • Clicar em "Painel Admin" para acessar a área restrita.
💡 Se o botão "Painel Admin" não aparecer, você pode não estar logado com o e-mail correto. Verifique o avatar no canto superior direito.

Aba "Provas Enviadas" — Gerenciar arquivos dos alunos

  • No Painel Admin, clicar na aba "Provas enviadas".
  • Use o filtro por prova para ver apenas os arquivos de uma prova específica.
  • Para cada arquivo enviado você verá: nome do arquivo, prova, quem enviou e data.
  • Clicar em "Baixar" para fazer download do arquivo e verificar a prova do aluno.
  • Clicar em "Excluir" para remover o arquivo do sistema (pede confirmação).
⚠️ A exclusão de arquivos é irreversível. Baixe o arquivo antes de excluir se precisar guardar uma cópia.

Aba "Rankings" — Gerenciar provas, inscritos e notas

  • No Painel Admin, clicar na aba "Rankings".
  • Ver a lista de todas as provas cadastradas com número de participantes e total de inscritos.
  • Na coluna "Inscritos", clicar em "Editar" para definir ou confirmar o total de inscritos daquela prova. Esse número é usado para calcular a posição estimada dos candidatos no concurso real.
  • Usuários logados podem sugerir o total de inscritos — mas a sugestão fica marcada como "não verificado" até que um admin confirme.
  • Clicar em "Ver ranking" para abrir o ranking completo de cada prova.
  • No ranking de uma prova, admins têm o botão "Excluir" em cada nota para removê-la se necessário.

Aba "Leads" — Captação de potenciais alunos

  • No Painel Admin, clicar na aba "Leads".
  • A lista exibe todos os usuários cadastrados que nunca marcaram "Sou aluno(a) OdonConcursos" em nenhum envio — ou seja, candidatos que ainda não são alunos.
  • Para cada lead você verá: nome, e-mail, data de cadastro e provas participadas.
  • Clicar em "Exportar CSV" para baixar a lista completa em planilha — útil para ações de marketing e abordagem comercial.
💡 Como os leads aparecem: Sempre que um candidato cria conta ou faz login, o perfil dele é salvo automaticamente no Firestore (coleção usuarios). A aba Leads filtra esses perfis comparando com o campo alunoOdon das notas enviadas.

Adicionar novo e-mail admin

  • Abrir o arquivo js/firebase-config.js na pasta do projeto.
  • Localizar o array ADMIN_EMAILS e adicionar o novo e-mail:
// Antes:
const ADMIN_EMAILS = [
  "michelecostaor@gmail.com",
  "adm.odonconcursos@gmail.com",
  "odonconcursosvendas@gmail.com"
];

// Depois (adicionando novo):
const ADMIN_EMAILS = [
  "michelecostaor@gmail.com",
  "adm.odonconcursos@gmail.com",
  "odonconcursosvendas@gmail.com",
  "novoemail@gmail.com"
];
  • Fazer a mesma adição no arquivo rules/firestore.rules dentro da função ehAdmin().
  • Salvar os dois arquivos e fazer o git push.
  • Republicar as regras no Firebase Console → Firestore → Rules com o novo conteúdo do arquivo.

Remover uma prova (ranking) do sistema

  • Acessar Firebase Console → Firestore.
  • Entrar na coleção provas e localizar o documento da prova a ser removida.
  • Dentro do documento, excluir a sub-coleção notas primeiro (se houver).
  • Em seguida, excluir o documento da prova.
⚠️ Não há como desfazer a exclusão de dados no Firestore. Tenha certeza antes de excluir.

4. Manutenção e Atualizações

Como atualizar o site e gerenciar o sistema

Como atualizar o site (qualquer arquivo)

Toda atualização segue o mesmo fluxo: editar o arquivo → salvar → enviar para o GitHub.

  • Editar o arquivo desejado na pasta do projeto no computador.
  • Abrir o Git Bash na pasta do projeto.
  • Rodar os comandos abaixo:
cd "C:/Users/chell/OneDrive/Área de Trabalho/ODONCONCURSOS/RANKING ODON ALUNOS"
git add .
git commit -m "descrição clara do que foi alterado"
git push
  • Aguardar ~1 minuto. O GitHub Pages atualiza automaticamente.
Dica: Use mensagens de commit descritivas. Ex: "Adiciona ranking da prova CFO 2026" em vez de só "update".

Quando precisa republicar as Firestore Rules

As regras do Firestore não sobem automaticamente com o git push — precisam ser republicadas manualmente sempre que o arquivo rules/firestore.rules for alterado.

  • Acessar: Firebase Console → Firestore → Rules
  • Apagar todo o conteúdo atual do editor.
  • Copiar e colar o conteúdo atualizado do arquivo rules/firestore.rules.
  • Clicar em "Publicar".
Quando isso é necessário? Só ao alterar as regras de acesso — por exemplo, ao adicionar/remover e-mails admin na função ehAdmin().

Gerenciar usuários (Firebase Authentication)

  • Acessar: Firebase Console → Authentication → Usuários
  • Ver todos os usuários cadastrados com e-mail, data de cadastro e último acesso.
  • Para desativar um usuário: clicar nos 3 pontos → "Desativar conta".
  • Para excluir um usuário: clicar nos 3 pontos → "Excluir conta".

Backup dos arquivos

Uma cópia de todos os arquivos do projeto foi salva na pasta BACKUP/ dentro da pasta de trabalho em 29/04/2026. Se algum arquivo for corrompido ou deletado por engano, pode ser restaurado a partir dessa pasta.

Recomendação: Sempre que fizer mudanças significativas no projeto, fazer uma nova cópia manual dos arquivos alterados para a pasta BACKUP.

5. Solução de Problemas

Erros comuns e como resolver

Problema: Site não atualiza após o git push

  • Aguardar mais 1-2 minutos — o GitHub Pages pode demorar um pouco.
  • Fazer um hard refresh no browser: Ctrl + Shift + R (Windows) ou Cmd + Shift + R (Mac).
  • Verificar em GitHub Actions se o deploy foi concluído (deve aparecer com ✅).

Problema: Erro "Permission denied" ou dados não carregam

  • Verificar se as regras do Firestore estão publicadas corretamente.
  • Acessar Firebase Console → Firestore → Rules e republicar.

Problema: Botão "Painel Admin" não aparece para um admin

  • Confirmar que o e-mail do usuário está exatamente como no array ADMIN_EMAILS no firebase-config.js.
  • Confirmar que o e-mail também está no firestore.rules.
  • Fazer logout e login novamente.
  • Se ainda não funcionar, verificar se o git push foi feito e se o GitHub Pages atualizou.

Problema: Erro no git push ("index.lock" ou "HEAD.lock")

  • No Git Bash, rodar: rm -f .git/index.lock .git/HEAD.lock
  • Tentar o push novamente.

Problema: Upload de arquivo não funciona

  • Verificar se o arquivo tem no máximo 10MB.
  • Formatos aceitos: PDF, JPG, PNG.
  • Verificar as configurações do Cloudinary no painel Cloudinary (upload preset odon-rankings-uploads deve estar ativo).
